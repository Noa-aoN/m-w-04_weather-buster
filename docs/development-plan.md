# 開発計画

## 方針

MVPは「遊べる最小単位」を先に通し、その後に天候ごとの個性を足していきます。

## マイルストーン

### Milestone 1: 開発基盤

- Vite + React + TypeScript の初期構築
- Three.js / React Three Fiber / Rapier 導入
- Lint / format の最小設定
- 初期ディレクトリ構成の確定

### Milestone 2: プレイヤー操作

- 3D空間の表示
- プレイヤー移動
- 視点操作
- ジャンプ
- ダッシュ
- 照準表示

### Milestone 3: 戦闘成立

- 射撃
- レイキャスト命中判定
- 敵耐久値
- プレイヤー耐候値
- 勝敗判定
- `CLEAR SKY!` / `WEATHER OVER`

### Milestone 4: 敵個性

- 曇り実装
- 豪雨実装
- 落雷実装
- 視界妨害や予兆の演出

### Milestone 5: ビルド要素

- 武器差分
- バスター固有スキル
- 武器スキル
- アイテム取得と使用

### Milestone 6: 結果画面

- 撃破時間
- 命中率
- 被ダメージ率
- 晴天化スコア
- ランク表示

## 直近の着手順

1. フロントエンドの土台を作る
2. 3D空間にプレイヤー視点を立てる
3. 射撃と単体敵を成立させる
4. 勝利演出として晴天化を入れる
5. 敵個性とスキルを追加する

## Definition of Done

各マイルストーンは、次を満たしたら完了扱いにします。

- 仕様との差分がドキュメント上で説明できる
- プレイ確認できる
- 主要な責務の置き場が崩れていない
- 次フェーズの前提が揃っている

## リスク

- 3D操作とFPS的な気持ちよさの調整は工数を食いやすい
- 天候演出を先に盛りすぎると、コア体験より演出実装が先行する
- 状態管理を雑に始めると、スキル・結果画面追加時に破綻しやすい

そのため、先に「移動」「射撃」「敵撃破」「晴天化演出」を最短で通します。
