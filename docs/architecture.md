# アーキテクチャ方針

## 目的

このプロジェクトでは、3D描画・ゲームループ・UI・永続状態を分離し、MVPから拡張段階まで破綻しにくい構成を取ります。

## 採用方針

- UIは React で管理する
- 3Dシーンは React Three Fiber 上で構築する
- 物理・当たり判定は Rapier を前提とする
- グローバル状態は Zustand に集約する
- ゲームルールは UI コンポーネントから分離する

## レイヤー構成

### `src/app`

- アプリ全体の起動点
- ルーティング
- グローバルプロバイダ
- 画面遷移の土台

### `src/scenes`

- タイトル画面
- 戦闘画面
- 結果画面
- ポーズ画面

画面単位の構成責務を持ちます。

### `src/game`

- ゲームループ
- 戦闘ルール
- ダメージ計算
- 勝敗判定
- スコア計算

ゲームロジックの中心です。

### `src/entities`

- プレイヤー
- 敵
- 武器
- アイテム
- 弾

ゲーム内オブジェクトごとの定義と振る舞いを持ちます。

### `src/features`

- 移動
- 射撃
- スキル
- アイテム使用
- HUD

横断的な機能単位を置きます。

### `src/shared`

- 定数
- 型
- ユーティリティ
- 汎用フック
- 共通UI

複数レイヤーで共有されるものを置きます。

### `src/assets`

- テクスチャ
- 音
- アイコン
- 一時アセット

## 状態管理方針

### Zustandで持つ状態

- 現在のシーン
- プレイヤー耐候値
- 敵耐久値
- 現在武器
- 残弾
- 気圧ゲージ
- タイマー
- スコア計算用メトリクス

### ローカル状態で持つもの

- 一時的なUI開閉状態
- 単一コンポーネントだけで完結する演出状態
- 一過性の入力表現

## データ定義方針

設定値は、可能な限りデータテーブルとして切り出します。

例:

- 敵の耐久値
- 武器の威力
- スキルの効果量
- アイテムの効果
- スコア係数

これにより、実装とゲームバランス調整の責務を分離します。

## MVP時点の責務境界

- `scenes`: 画面の並びと見た目
- `game`: 戦闘進行と判定
- `entities`: オブジェクト単位の振る舞い
- `features`: 入力やHUDなどの機能組み立て
- `shared`: 型・定数・補助関数

UIが直接ダメージ計算や勝敗判定を持たない構成を維持します。

## 将来拡張を見据えた注意点

- 現在地天気連動は `game` ではなく外部データ層として切る
- ランキングはMVP外なので、スコア計算だけ先に分離する
- 台風のような複合ボスを想定し、敵挙動はコンポジション可能にする
- 視界妨害や鈍足などの状態異常は共通ステータスとして扱う
