# ディレクトリ構成

## 目的

実装前に、ファイルの置き場と責務を固定して、途中で構成が崩れるのを防ぎます。

## ルート構成

```txt
.
├── README.md
├── docs/
├── public/
├── src/
└── tests/
```

## 各ディレクトリの責務

### `docs/`

- 仕様
- 設計
- 開発計画
- 紹介文

### `public/`

- favicon
- 画像
- そのまま配信する静的ファイル

### `src/`

- アプリケーション本体
- ゲームロジック
- UI
- 3Dシーン

### `tests/`

- 将来的なユニットテスト
- ロジック検証
- シナリオベースの確認項目

## `src/` の初期構成

```txt
src/
├── app/
├── assets/
├── entities/
├── features/
├── game/
├── scenes/
└── shared/
```

## 命名方針

- 画面単位は `Scene` サフィックスを使う
- 3Dオブジェクト単位は責務名をそのまま使う
- Zustandストアは `useXxxStore` に統一する
- 定数ファイルは `constants.ts`
- 型定義は `types.ts` または対象名 + `Types.ts`

## 運用ルール

- 仕様追加時は、先に `docs/` を更新する
- 画面を増やす時は `scenes/` を起点に追加する
- 戦闘ルールを増やす時は `game/` を起点に整理する
- 汎用化前提で早く `shared/` に逃がしすぎない

MVPでは過剰な抽象化を避け、責務が明確なまま拡張できることを優先します。
