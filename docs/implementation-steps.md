# 実装手順

## 目的

この文書は、MVPに向けた実装順と、開発者が手作業で確認・判断する作業を分けるための作業メモです。

## 実装の進め方

### Step 1: 開発基盤

実装内容:

- Vite + React + TypeScript を導入する
- Three.js / React Three Fiber / Drei / Rapier / Zustand を導入する
- `src/` の責務分割に沿って初期ファイルを配置する
- 開発サーバーで画面が表示できる状態にする

手作業で確認すること:

- `pnpm install` が成功するか
- `pnpm dev` でローカル表示できるか
- ブラウザで画面が真っ白になっていないか

### Step 2: ゲーム画面の骨格

実装内容:

- フルスクリーンの3Dシーンを表示する
- 実験場ステージの床・グリッド・障害物を置く
- HUDを重ねる
- 敵天候を1体表示する

手作業で確認すること:

- 3Dシーンが画面全体に表示されるか
- HUDが3Dシーンと重なっても読めるか
- PCとスマホ幅でUIが崩れないか

### Step 3: 最小戦闘

実装内容:

- 左クリックで射撃する
- 射撃が敵に命中した扱いになる
- 敵耐久値を減らす
- 敵耐久値が0になったら `CLEAR SKY!` を表示する

手作業で確認すること:

- クリック時に撃った感があるか
- 敵耐久値が減っていることがUIで分かるか
- 勝利演出が気持ちよく見えるか

### Step 4: プレイヤー操作

実装内容:

- `WASD` 移動
- `Shift` ダッシュ
- `Space` ジャンプ
- マウス視点操作

手作業で確認すること:

- 移動速度が速すぎないか
- ダッシュが戦闘中に使いやすいか
- 視点移動で酔いにくいか

### Step 5: 敵ごとの個性

実装内容:

- 曇り: 練習用の弱い敵
- 豪雨: 雨弾と水たまり
- 落雷: 落雷予兆と高威力攻撃

手作業で確認すること:

- 敵ごとの差がプレイ中に分かるか
- 攻撃予兆が理不尽に見えないか
- 初期MVPとして難しすぎないか

### Step 6: スキルとアイテム

実装内容:

- バスター固有スキルを常時パッシブとして反映する
- 命中で気圧ゲージを増やす
- `Q` で武器スキルを発動する
- フィールドアイテムを拾い、`E` で使う

手作業で確認すること:

- ゲージが溜まる理由が分かるか
- スキル発動タイミングに判断の余地があるか
- アイテムが強すぎないか

## 今回の初回実装範囲

今回の初回実装では、以下までを対象にします。

- Vite + React + TypeScript の導入
- 3Dシーンの表示
- HUDの表示
- 敵天候1体の表示
- クリック射撃による敵耐久値の減少
- `CLEAR SKY!` の表示

プレイヤー移動・敵攻撃・物理演算は、次の実装ステップで追加します。

## 手作業でやること

- 初回は `pnpm install` を実行する
- 実装後に `pnpm dev` で画面確認する
- ブラウザでクリック操作と表示崩れを確認する
- ゲーム体験として「晴れた感」が弱い場合は、演出の方向性を決める

## コマンド

```sh
pnpm install
pnpm dev
pnpm build
```
