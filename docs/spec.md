# 仕様・設計判断

## アプリ概要

**ウェザーバスター 〜CLEAR THE SKY〜** は、異常気象モンスターを撃破して荒れた空を晴れに戻す3Dアリーナシューターです。

MVPでは「敵天候を倒す」「空が晴れる」「`CLEAR SKY!` が表示される」体験を最優先で成立させます。MVP 後の拡張段階では、武器・パイロット・戦域・敵の組み合わせで戦術が変わるバリエーション構成を採用します。

## MVP範囲

- 実験場ステージ
- 敵天候: 曇り、豪雨、多雷
- 武器: ウェザーガン、クリアスカイガン、梅雨キラー
- スキル: キャラ固有スキル、武器スキル
- アイテム: 晴れ薬、デコイ傘、避雷針、気圧安定剤
- 結果画面: 撃破時間、命中率、被ダメージ率、晴天化スコア、ランク

## 画面遷移

- ホーム（タイトル / メニュー / ロードアウト要約 / 戦域・敵の即時切替）
- ストーリー（導入の章立てテキスト）
- 観測記録（敵図鑑、出撃選択）
- キャラ（パイロット図鑑。3D プレビュー付きで選択）
- 装備（武器 / パイロット / 戦域の3タブ。ホームから該当タブへ直接遷移）
- 設定（マウス感度 / FOV / 照準カラー / 音響）
- 戦闘（選択された戦域）
- 結果（ランク・統計）

ホームが起点。各サブ画面はホームから直接遷移し、`H` または `Esc` でホームに戻る。戦闘は装備済み武器・パイロット・戦域と選択中の敵で開始する。戦闘終了は結果画面に遷移し、結果からはタイトル復帰または再観測が選べる。

各画面は遷移時に短いフェード/スケールアニメーション（`sceneEnter`）で揃え、ヘッダー（PROJECT / 画面名 / 戻る）とボタンの hover/active/focus を共通化する。

### ホームの導線設計（C案ハイブリッド）

軽量変更（戦域・敵）と深層変更（武器・パイロット・戦域詳細）を分離する:

- 戦域 / 敵: MissionPreview の ◀ ▶ ボタンで即時切替（ホームから出ない）
- パイロット / 武器 / 戦域詳細: ロードアウト要約カードを押すと該当タブに直接遷移
- 全体的な装備見直し: メニューの「装備」ボタンから WEAPON タブで開く
- 設定 / 観測記録: メニューから単独画面を開く

これにより、リプレイ時は ◀ ▶ + ゲーム開始の最短導線、初回や本格的なビルド変更は要約カード経由の深層導線、というふうに利用シーンに合わせた使い分けができる。

## 位置情報連動

ホームヘッダの右上（ONLINE の左）に GPS トグルを配置する。ON にすると Geolocation API で現在地を取得し、Open-Meteo（API キー不要）で現在の天候を取得して、対応する敵を自動選択する。

天候コードのマッピング:

- 0 (快晴): 自動選択なし
- 1-3, 45/48: 曇り
- 51-67, 80-82: 豪雨
- 71-77, 85-86: 吹雪
- 95-99: 多雷

権限拒否やネットワーク失敗時は静かにトグルを OFF に戻す。プライバシーのため、座標は保存せずトグル状態だけを保持する。

## 操作

戦闘中:

- `WASD`: 移動
- `Mouse`: 視点
- `Left Click`: 射撃（命中はカメラからのレイキャストで判定）
- `Right Click` / `R`: リロード（**必須**。空マガジンの自動リロードは廃止）
- `B`: 気象シールド（押している間だけ展開）
- `Space`: ジャンプ
- `Shift`: ダッシュ
- `Q`: 武器スキル発動（気圧ゲージ満タン時）
- `1`〜`4` / `E`: アイテム使用（1=晴れ薬, 2=避雷針, 3=デコイ傘, 4=気圧安定剤、`E` は晴れ薬の別キー）
- `Esc`: ポインタロック解除（ポーズ）

ホームメニュー:

- `Enter`: ゲーム開始
- `T`: ストーリー
- `G`: 観測記録（ENEMY GRID）
- `C`: キャラ（PILOT GRID）
- `L`: 装備（LOADOUT）
- `S`: 設定
- `←` / `→`: 敵の前後切替
- `[` / `]`: 戦域の前後切替

サブ画面共通:

- `H` / `Esc`: ホームへ戻る

メニュー / バナー:

- `Enter`: メイン操作（戦闘開始 / プレイ続行 / 結果を見る / 再観測）
- `H`: ホームへ戻る（ready / pause / clear / defeat / result バナー共通）
- `R`: 再観測（result バナー）
- 各ボタンには対応キーを括弧内表記（例: `戦闘開始 (Enter)`, `← ホーム (H)`）

音響共通:

- `M`: SFX / BGM をまとめてミュート切替
- `,` / `.`: マスターボリュームを 0.1 単位で増減

## アイテム効果

- 晴れ薬: 耐候値を 350 回復する
- 避雷針: 場のすべての落雷予兆を消す
- デコイ傘: 短時間（5秒）被ダメージを 60% 軽減する
- 気圧安定剤: 気圧ゲージを 60 加算する

## 武器スキル

各武器に固有のスキルを持つ。命中で気圧ゲージが溜まり、満タン時に Q で発動する。武器スキルの威力はキャラのパッシブ倍率と敵相性倍率の積。

- ウェザーガン / 天候連射: 連射で削る短期バースト（バランス）
- クリアスカイガン / 晴天バースト: 単発高威力（バランス）
- 梅雨キラー / 湿度排除: 豪雨/梅雨へ追加ダメージ
- 雷除けライフル / 電界反射: 多雷へ追加ダメージ
- 凍結カノン / 凍結弾: 吹雪/竜巻へ追加ダメージ

## パイロット（キャラ固有スキル）

3名のパイロットから1名を選択する。スキルは常時発動パッシブ。

- アヤメ / IRIS / 晴天適応: 与ダメージ +10%
- ハル / HALO / 気圧耐性: 被ダメージ -18% / 与ダメージ -5%
- ライカ / RAIKA / 気圧収束: 気圧ゲージ +30% / 被ダメージ +10%

## 戦域（ステージ）

3戦域から1つを選択する。配色・霧・スカイに加え、`stage.arena` で可動範囲を、地形は名前に応じて専用コンポーネント（`StageTerrain`）で構築する。

- 実験場 (lab): シアン基調の整備された円形フィールド。アリーナは ±11、地形は試作機検証用に整備された矩形ブロックの並び。
- 都市跡 (ruins): 橙赤の崩壊した観測都市。アリーナは ±19。瓦礫・傾いたタワー・残った柱で非対称な地形を構築する。
- 高地観測所 (highland): 淡い氷青の高地。アリーナは ±21。外周に雪冠の山稜、内側に雪化粧した観測プラットフォームと岩塊、奥に観測ドームを置く。

## 難易度

各敵に `difficulty: 1〜5` を持たせ、HP・敵攻撃ダメージ・攻撃間隔・所持アイテム数・移動の積極性を一括で調整する。MISSION PREVIEW では 5段階の階段表示と EASY/NORMAL/TOUGH/HARD/EXTREME のラベルで提示する。

| 難易度 | HP倍率 | 攻撃倍率 | 攻撃間隔 | アイテム | 動きの激しさ |
| --- | --- | --- | --- | --- | --- |
| 1 (EASY) | 0.7 | 0.55 | 1.6x | 1.6x | 0.5x |
| 2 (NORMAL) | 0.9 | 0.85 | 1.25x | 1.3x | 0.75x |
| 3 (TOUGH) | 1.0 | 1.0 | 1.0x | 1.0x | 1.0x |
| 4 (HARD) | 1.25 | 1.25 | 0.82x | 0.7x | 1.3x |
| 5 (EXTREME) | 1.6 | 1.55 | 0.65x | 0.5x | 1.7x |

## 敵 AI と攻撃演出

敵は `EnemyMotion` で5つのフェーズ（approach / circle / telegraph / retreat / idle）を回す。circle ではプレイヤー周辺を半径6.2-9前後で旋回し、orbit 方向はランダムに反転する。telegraph では一瞬距離を詰めて攻撃の予兆を見せる。idle はフェーズ遷移時に確率で発生する動作緩慢区間で、移動・上下動が両方鈍る（曇り 32% / 梅雨 26% / 他 18%）。竜巻・台風は常に y 軸回転を伴い、垂直方向の浮き沈みも大きい。

プレイヤーとの距離は最低 5.5 を保ち、approach / retreat の目標位置は常にプレイヤーから見て一定距離（standoff = 7.5）になるよう極座標で再計算する。これにより敵が密着して張り付くことを避ける。

攻撃の種類は `enemyAttackPatterns[id].kind` で3種:

- `arc`（曇り / 多雷 / 竜巻 / 梅雨 / 吹雪）: 敵位置から放物線で着弾点へ飛翔
- `linear`（豪雨 / 台風）: 敵位置から地表ターゲットへ直線飛翔。短い警告で連射型
- `falling`（多雷）: 真上 ~14 の高所から着弾点へ垂直落下。雷撃のメタファ

`MarkerProjectile` は kind に応じて軌道計算とメッシュ形状（球 / カプセル）、回転・スケールを切り替える。

攻撃は `enemyAttackPatterns` で敵ごとに調整した間隔・警告時間・ダメージ・着弾半径・追従可否で発生する。スポーン時に敵位置を起点として `AttackMarker` を作成し、

- 警告リング（地面）: 警告時間に応じて発光が強まる
- プロジェクタイル: 敵の現在位置から着弾点へ放物線を描いて飛翔し、軌跡が視認できる発光球＋ポイントライトを伴う
- 着弾: `triggersAt` 時点でプレイヤーが半径内なら `takeMarkerDamage`

これによって、攻撃に対して「いつ／どこへ」「何が飛んできて」「何秒後に当たるか」を視覚的に判断できる。

## 弱点（コア）

各敵モデルの中央には `enemyCore` という名前と `userData.isCore = true` を持つ発光球を配置する。射撃のレイキャストでヒットしたオブジェクト系列にこのフラグが含まれる場合はクリティカルヒットとして扱い、ダメージ x2.4・気圧ゲージ加算量を上乗せする。

## 敵相性

武器ごとに `specialtyAgainst` と `specialtyMultiplier` を持ち、対象敵に対するダメージを補正する（通常射撃・武器スキル両方に乗る）。

ホームのミッションプレビューでは、選択中の敵に対して武器が `specialtyAgainst` に含まれていれば「×N.NN 弱点特効」バッジが表示される。

## 敵バリア / 攻撃ノックバック

各 `enemyAttackPatterns` には次の 3 つの追加プロパティがある:

- `knockback: number` ─ 着弾時、プレイヤーをインパクト中心から離れる方向に押し出す強さ（0〜1.6）
- `barrierIntervalMs: number` ─ 0 でないとき、その間隔ごとにバリアを起動
- `barrierDurationMs: number` ─ バリアの持続時間

代表値:

| 敵 | knockback | barrier (ms 間隔 / 持続) |
| --- | --- | --- |
| cloudy | 0 | なし |
| heavyRain | 0.6 | なし |
| thunderstorm | 0.4 | 17000 / 1800 |
| tornado | 1.6 | なし |
| rainySeason | 0.3 | なし |
| blizzard | 0.8 | なし |
| typhoon | 1.4 | 14000 / 2400 |

バリア中の射撃は **ダメージ x0.18 / 気圧ゲージ x0.4 / クリ判定無効化** になり、HUD に「BLOCKED」と「BARRIER UP」が表示される。
ノックバックは `applyKnockback(vx, vz)` で速度を蓄積し、`PlayerController` の `useFrame` 内で `consumeKnockback` を毎フレーム呼んで 14% ずつ減衰させながらカメラ位置に積分する。

## 敵スペシャルアタック

通常攻撃に加え、敵 ID ごとに次の周期で 3〜5 マーカーをプレイヤー周辺の円周上に展開するスペシャルが発動する:

| 敵 | 周期 (s) | 発数 |
| --- | --- | --- |
| typhoon | 14 | 5 |
| heavyRain | 16 | 5 |
| thunderstorm | 17 | 4 |
| rainySeason | 19 | 4 |
| blizzard | 20 | 3 |
| tornado | 21 | 3 |
| cloudy | 26 | 3 |

各マーカーは標準の 半径 ×1.05 / 警告時間 ×1.2 / ダメージ ×1.15 / trailGlow ×1.4 と少し強化される。
スペシャル発動 1.2 秒前に `enemyChargeStartedAt / enemyChargeFiresAt` がストアに立ち、敵が発光球と二軸 torus で予告（`EnemyChargeFx`）。HUD には「SPECIAL INCOMING」プログレスバーが出る。発動時は `playEnemyChargeFire`（60Hz サブベース + 連続ノイズ）が鳴る。

## ランク

結果画面のランクは、スコア・命中率・被ダメージ率の組合せで決定する（S/A/B/C/D）。クリアできなかった場合は D 固定。

結果画面では同じ敵 × 難易度の `seedHistory` と比較して、初回クリアなら「FIRST CLEAR」、過去ベスト更新なら「NEW BEST」のバッジが表示される。

## 弾薬とリロード

各武器は短いマガジンと専用リロード時間を持つ:

| 武器 | maxAmmo | damage | reload (ms) |
| --- | --- | --- | --- |
| ウェザーガン | 32 | 11 | 950 |
| クリアスカイガン | 22 | 17 | 1100 |
| 雷除けライフル | 14 | 16 | 1300 |
| 凍結カノン | 10 | 24 | 1700 |
| 梅雨キラー | 9 | 28 | 1700 |
| 風向ブレード | 6 | 36 | 1700 |

リロード時間は `computeReloadMs(weapon)` で `maxAmmo` のしきい値ごとに決まる。`R` キーで手動リロード、マガジン空時は自動でリロードを開始（プレイヤーの操作ストレスを軽減）。リロード中は `shoot()` がガード節で発射不可になり、画面下中央に RELOADING プログレスバーが表示される。

## 晴天の種子（メタ進行）

`useBattleStore` が `seedCount` と最大 32 件の `seedHistory` を保持し、`localStorage("weatherbuster-seeds-v1")` に永続化する。クリア毎に **難易度値（1〜5）** 分の種子が加算される。

ホーム左中段の「晴天の種子」カード（`SeedCounter`）に総種子数 + 戦闘記録件数 + 直近 4 戦の (ランク・敵名・難易度) ミニリストが並ぶ。ストーリーで掲げた「晴天の種子を集めて空を恒久的に取り戻す」目標と直結する。

## HUD UX

- 着弾フラッシュ（hit marker）: 命中で敵アクセント色 / 外しで白の十字フラッシュ
- 弾道トレーサー: 発射時に銃口から前方へ伸びる発光ボックスが瞬時に走り、フェードする
- 銃口フラッシュ: 発射と同時に銃口先端で黄白色の発光球とポイントライトが80msフラッシュ
- 武器スキル発動: 全画面の黄色パルス
- アイテム使用: 上部にトーストで効果を表示
- 照準カラー: 設定画面のプリセットから即時反映
- ポーズメニュー: Esc 解除時に「プレイ続行 / 撤退してタイトルへ」を表示
- 撃破演出: 敵HP0で 360ms 膨張 → 900ms 縮小フェード、同時に24粒の球状パーティクルが弾けて減衰する
- ESC ヒント: 戦闘中、画面上部中央に「ESC マウス操作に戻す（ポーズ）」のミニバーを常時表示する
- ホーム背景: タイトル画面の地形・機体・観測装置はゆっくり (約 0.045 rad/s) 公転する。降水・雷光は世界座標のまま据え置く

## 音響

WebAudio API のオシレータ + ノイズバーストで効果音と BGM を合成する（`src/features/audio/audio.ts`）。

- SFX: 射撃 / ヒット（通常・クリティカル）/ ミス / 武器スキル / アイテム使用 / 攻撃マーカーのスポーン / 着弾 / クリア / ディフィート / カウントダウン（チック・GO）
- BGM: シーン切替式のステップシーケンサ。バス・リード・パッド・ドラム（キック / スネア / ハイハット / オープンハット）の 4 トラックを 16 / 32 ステップでループ
  - `title`: C メジャー進行（C-G-Am-F）/ 110 BPM。ホーム・ロードアウト・ストーリー等の準備画面
  - `battle`: A マイナー進行（Am-F-C-G）/ 138 BPM。ロックビートとペンタトニックのリードで戦闘の高揚感を作る
  - `victory`: D メジャーのファンファーレ / 120 BPM。CLEAR SKY 演出と同期
  - `defeat`: D マイナー下降フレーズ / 70 BPM。重く沈む雰囲気
- BGM のシーン切替は `setBgmScene(scene)` で行い、`AudioBridge` が `useBattleStore.status` を購読して自動同期する
- 状態は `useBattleStore` の `sfxEnabled / bgmEnabled / masterVolume` で保持し、`AudioBridge` が音響モジュールに同期する
- 右上の `AudioToggle` で SFX/BGM の切替とマスターボリュームを操作。`M` キーで両方ミュート、`,` `.` キーで音量 ±0.1
- 設定画面（SETTINGS）にも同等の SFX/BGM トグルとボリュームスライダーを置く

## 戦闘開始演出

`戦闘開始 (Enter)` 押下時、`status` を `ready` から `battle` に切り替える前に 3 → 2 → 1 → GO! のカウントダウン演出を挟む（合計 2.7 秒）。

- 各カウント時に短いチック音（高音 triangle / sine）、GO! で広域フラッシュとノイズバーストを鳴らす
- 中央には大きな数字とリングのモーション（`countdownPop`, `countdownRing`）が出る。GO! はカラーリングを緑系に切り替え少し大きく弾ける
- カウントダウン中は新しい開始操作を無視する。`useEffect` のクリーンアップでタイマーを破棄する

## 命中フィードバック / 画面演出

- ダメージ数値ポップアップ: 命中時に画面中央付近へ与ダメージを表示。クリティカルは 1.5 倍サイズ + `CRIT!` 表記 + 強グロー
- コンボカウンタ: 連続命中数を画面右上に表示。最後の命中から 2.4 秒以上空く / 外す / バトル終了でリセット。HP・与ダメージ等の数値補正は行わず、見た目だけの演出
- スクリーンシェイク: クリティカル / シールドブロック / スキル発動 / クリア / ディフィートで `.gameShell` を `:has()` 経由で揺らす（light: 0.22s 1px〜3px、heavy: 0.5s 4px〜7px + 微回転）
- 低 HP ヴィネット: HP 比 32% 未満で赤い radial グラデーション。18% 未満で更に強くなり高速パルス
- バトル開始フラッシュ: `status === "ready"` から `"battle"` に遷移した瞬間、画面中央から黄白色の radial フラッシュを 0.6s 走らせる
- スキル発動演出: 全画面の skillFlash + 同心円 3 段の skillBurstRing（時間差で展開）+ スキル名バナーが上部 38% に飛び出して消える
- アイテム使用トースト: 上部 80px に大型アイコン + 名前 + 効果のトーストを 1.8s 表示。アクセント色はアイテムごとに分岐（`color-mix` で発光色を作る）
- 被ダメージフラッシュ: HP が下がった瞬間、画面端から赤いラジアルが差し込む。下落量に応じて強度を変化（最大 0.48s）
- ヒールフラッシュ: `clearTonic` 使用時、画面端から黄緑系の柔らかいラジアル（0.7s）
- 敵リビールアニメ: `status === "ready"` から `"battle"` に遷移した直後、敵スケールが 0 → 1.55 へ 0.7s で立ち上がる（オーバーシュート付き）
- HitMarker / SkillFlash 元実装 / 銃口フラッシュは従来どおり

## 敵 3D デザイン

`WeatherEnemyModel` は敵 ID ごとに表情とパーツを変えた専用コンポーネントを返す。

- 共通: `enemyCore` 名 / `userData.isCore = true` の発光球を中心 `(0, 0.05, 0.15)` に配置（クリティカル判定用）。回転するアクセントカラーのハロ（torus）を一周
- `cloudy`: 5 ピース雲 + クリッと丸い目 + 小さな口 + 頬の赤らみ
- `heavyRain`: 7 ピース雲 + 怒り眉の目 + しょんぼり口 + 連続落下する 4 本の雨カプセル
- `thunderstorm`: 7 ピース雲 + 怒り眉 + ギザ口 + 左右に伸びる雷円錐（ちらつき）
- `tornado`: 4 段の傾いた torus + 頭部の球 + ぱっちり目 + 開いた口
- `rainySeason`: 7 ピース雲 + 雨カプセル + 足元の波紋 ring
- `blizzard`: 7 ピース雲 + 閉じ目 + 回転する雪の結晶 (torus)
- `typhoon`: 5 段 torus + 大きい頭 + 怒り眉 + 底部の発光リング
- 撃破時 (`status === clear`) は表情が笑顔（`smile` 口）になり、雲は白系で発光、雨や雷は消える

## ポーズと進行管理

`status === "battle" && isPointerLocked` を「アクティブ戦闘」の唯一の真とする。Esc 等でポインタロックが外れた瞬間にポーズ扱いとなり、次の更新が止まる:

- `tick()` `takeDamageTick()` (1秒 / 3秒インターバル) はガード節で早期 return
- 攻撃マーカーのスポーン (`PlayerController` useFrame) と着弾判定 (`takeMarkerDamage`) も同条件で停止
- `EnemyMotion` のフレーム更新も停止（敵は静止する）
- 既存マーカーの `triggersAt` は絶対時刻のため、ポーズ → 再開の差分時間を `shiftMarkerTimes` で全マーカーへ加算し、再開直後の即着弾を防ぐ。次回攻撃時刻 (`nextLightningAt`) も同様にシフト

ホームへの戻り（`returnToHome`）は常に `reset()` を呼び、再度「ゲーム開始」した際は HP・弾薬・アイテム・撃破フラグを初期状態に戻す。戦闘中の battleBrand 押下や撤退ボタンも `returnToHome` 経由で初期化される。

## バランス調整指針

- プレイヤー耐候値（HP）は 1000。武器ダメージ・敵攻撃ダメージの数値感に合わせ、被弾10数発でディフィートに至る想定。
- 敵 HP のベース値はおおよそ 240〜1100 で、武器ダメージ（9〜22）に対して数十発レンジで戦闘が進むよう調整する。
- アイテム所持数は難易度倍率を `Math.floor` で適用するため、難易度 5 では一部アイテム（気圧安定剤など）が 0 個になり得る。これは仕様。

## 技術選定

- Vite: 3Dゲーム試作に必要な軽い開発サーバーとビルド環境を短く作れるため
- React: HUD、画面遷移、状態表示をコンポーネントとして扱いやすいため
- TypeScript: 敵・武器・スキルなどのゲームデータを型で管理するため
- Three.js / React Three Fiber: React の構成に合わせて3Dシーンを実装するため
- Drei: カメラ、Sky、補助コンポーネントを早く扱うため
- Rapier: 今後の移動、当たり判定、物理挙動に備えるため
- Zustand: 戦闘状態、HUD表示、スコア計算用メトリクスを軽量に共有するため

## パッケージマネージャ

このアプリでは `pnpm` を使います。

理由:

- lockfile を1種類に統一する必要があるため
- 兄弟プロジェクトで pnpm 運用実績があり、ローカル環境にも導入済みのため
- Vite / React / Three.js の構成で npm 固有の必要性がないため

## 初回実装判断

最初の実装では、物理演算や本格的なFPS移動より先に、3Dシーン・HUD・射撃・敵HP・`CLEAR SKY!` 表示を通します。

理由:

- MVPの核は「晴天化の達成感」だから
- 移動や敵AIより前に、勝利演出の方向性を確認できるから
- 以降の調整対象を、プレイヤー操作と敵個性に分離できるから
